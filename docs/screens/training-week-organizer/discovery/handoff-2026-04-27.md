Marco trains in three concurrent shapes and they were getting lost between Notion, his physio prescription, and his head. **From the Ground Up** is a 6-week physio prescription — every exercise, every day, 1 set each. **ABCDE** is a gym rotation — one workout per day Mon→Fri at the gym. **Snacks** are 5–15 min micro-sessions thrown in opportunistically (cranium release, pelvic floor, neck, DNS prone/supine). The feature gives him: a workout library, multi-session days (multiple sessions on the same date), a week planner with a themed intent and focus areas, a weekly review, and post-session sensation capture distinct from RPE.

## 4. Architectural decisions already made (don't relitigate)

These were settled with Marco. If you think one is wrong, flag it before changing — don't silently rework.

| Decision | Rationale |
|---|---|
| Three new collections: `protocols`, `workouts`, `weekPlans` (NOT extending `programs`) | `programs` is a 4-week macro plan. Workouts are reusable templates; weekPlans are themed single-week schedules; protocols are time-bounded prescriptions. Different lifetimes, different shapes. |
| `From the Ground Up` is a `Protocol` with `startDate`, `endDate = startDate + 42 days`, `workoutIds[]`, `status: active` | Time-bounded; auto-flips to `completed` after `endDate`. Today screen pins it as a persistent daily card with X-of-Y progress. |
| ABCDE auto-fills Mon→Fri (`day=1` → first abcde workout, etc.) on first read of a `weekPlans/{isoYearWeek}` doc | Slot defaults to `afternoon` (typical gym time). User can drag to override. `autoAssigned: true` flag distinguishes auto vs manual placement. |
| Multi-session days via `Session.slot` (morning / midday / afternoon / evening / flexible) — NOT timestamps | Calendar UX is cleaner with named slots; timestamps come later if real time-of-day matters. |
| `ExerciseBlock` is the same value object inside `Workout` (template) AND `Session` (instance). Templates leave `actualSets` empty; sessions populate. | Avoids parallel `WorkoutBlock` / `SessionBlock` classes that drift apart. |
| Notion's "Current Included" flag → `ExerciseBlock.currentlyIncluded` (default true). False = parked for later (Strength / Integration ladder). `Workout.activeBlocks` and `inactiveBlocks` getters split them. | Marco can grow into Strength tier without losing the parked entries. |
| `SensationFeedback` is a separate value object, NOT folded into Session top-level | RPE is effort. Sensation is body-awareness. They're distinct. Keep `bodyFeeling` in web-recorder as the short check-in; this is the long one. |
| Notion-import is **idempotent** via `idempotencyKey: "notion:<notion-id>"` per doc | Re-running the import is safe. |
| Behind feature flag `feature_training_week_v2` in Firebase Remote Config | Default `false` until E2E green. |
| Web target = **Flutter Web** (`flutter run -d chrome`), NOT React. The `web-recorder/` Svelte sibling is camera-only and out of scope. No `frontend/web/` React app — that's blocked by `.claude/rules/web_frontend/architecture.md` until Phase 3. | |

## Quality bar to keep up

All 14 screens shipped this pass follow the same rules — the next agent should hold to them.

### Theme tokens — never hardcode

- Colors come from `lib/core/theme/app_colors.dart`. Key ones the revamp leans on:
  - `AppColors.primary` (terracotta, CTAs, active/selected states)
  - `AppColors.accent` (sage, body-awareness confirmation, "good" states — **never CTAs**)
  - `AppColors.reward` (soft gold — milestone/achievement only, rare)
  - `AppColors.severityResolved/Improving/Mild/Moderate/Significant` — the 5-step severity ramp for compensations & detected patterns
  - `AppColors.textOnPrimary`, `AppColors.textSecondary`
  - Dark-mode surface tokens (`backgroundDark`, `surfaceDark`, `surfaceRaisedDark`, `borderDark`) flow through the theme — prefer `theme.scaffoldBackgroundColor` / `theme.colorScheme.surface` / `theme.colorScheme.outline` over referencing the dark tokens directly.
- Spacing/radii come from `lib/core/theme/app_spacing.dart`. Use `AppSpacing.{xs,sm,md,lg,xl,xxl}` for padding and `AppSpacing.{radiusSm=10, radiusMd=14, radiusLg=20, radiusXl=28}` for corners. Cards are `radiusMd`; modals/bottom-sheets are `radiusXl`.
- Motion tokens live in `lib/core/theme/app_motion.dart`. Use `WayMotion.micro` (120ms) for taps, `WayMotion.standard` (280ms) for navigation/reveal, `WayMotion.settled` (450ms) for completion celebrations, `WayMotion.breath` (2400ms) for the journal-mic listening loop. Curves: `WayMotion.easeMicro/easeStandard/easeSettled/easeBreath/easeReward`.
- Typography: `AppTypography.fraunces(...)` for display/hero numbers/journal moments (often italic); `AppTypography.manrope(...)` for all UI. For normal body/headline usage prefer `theme.textTheme.displaySmall|headlineMedium|titleMedium|bodyMedium|labelSmall` — the theme already wires those to the right font and color.

### Non-negotiables seen on every screen this pass

- **Min tap target 48px** (`AppSpacing.minTapTarget`). Primary CTAs are 56px.
- **Underline-only inputs.** The theme already configures `InputDecorationTheme` that way — **never** pass `border: OutlineInputBorder()` or `filled: true` on inputs unless there's a very specific reason.
- **Buttons accept the theme.** Don't re-specify `backgroundColor: AppColors.primary` on `FilledButton` — the theme handles it. The exception in this pass is the "Mark as achieved" button on Goal Detail which overrides to sage because sage is the semantic "confirmation" color there.
- **Page transitions animated.** Use `WayMotion.easeSettled` for entry fades, `WayMotion.easeStandard` for standard transitions. Don't leave raw `Curves.easeOut`/`Duration(milliseconds: 300)` behind.
- **No drop shadows** except on pinned bottom CTAs (see Screen 6's `_BottomBar`). Cards rely on borders, not elevation.
- **Sage is never a CTA.** If you find yourself reaching for sage on a primary button, you're probably in a "confirmation" context (Goal Detail's achieve button is the one exception — it's a confirmation *act*).

### Per-screen patterns this pass established (reuse where they fit)

- **Fraunces display app-bar title** with `toolbarHeight: 72` for top-level screens (Exercises, Goals, Body awareness, etc.).
- **Fraunces italic question/prompt** as the centerpiece when a screen is a "single calm moment" (Assessment question steps, Journal Entry, Sleep Log body title).
- **Custom-painted score/progress rings** using a `CustomPainter` — see:
  - `_ScoreRingPainter` in `initial_assessment_flow.dart` (terracotta→sage sweep gradient)
  - `_HeroRingPainter` in `goal_detail_page.dart` (solid)
  - `_RingPainter` in `goal_list_page.dart` (solid, small)
- **4px terracotta left strip on "current" items** — see `session_view.dart` `_ExerciseBlockCard` and `initial_assessment_flow.dart` `_OptionCard`. Implemented via `Stack` + `Positioned.fill(width: 4)` to dodge the `Border`+`borderRadius` asymmetry constraint.
- **Severity bar component** — see `_PatternTile` in `initial_assessment_flow.dart` and `_CompensationCard` in `ai_recommendation_review_page.dart`. Use `ClipRRect` + `Container(color: outlineVariant)` + `FractionallySizedBox` with a `color: severityX`.
- **Sage tag pill (outlined)** — see `_TagPill` in `exercise_card.dart`. 1px border of `AppColors.accent` at 0.45 alpha, sage text.
- **Progress dots (stretching)** — see `initial_assessment_flow.dart` `_buildHeader`. The "current" and "passed" dots grow from 8→24px wide; inactive stay 8×8.

### Testing discipline

- Every widget change must leave the existing widget tests green. When the plan implies a text change (e.g. "Start Assessment" → "Begin") but the existing test asserts on the old copy, **keep the old text**. We optimize for zero regression; content polishing can be a follow-up.
- If you add a brand-new page surface (happened in this pass for Exercise Detail), add one small widget test that checks the page key, the page title text, and at least one interactive affordance — don't ship a new surface with zero coverage.
- Keys that other tests reference live in `lib/core/constants/app_keys.dart`. **Don't** remove keys; move them if you restructure.

### Analyze discipline

- Run `flutter analyze <path/to/just-edited-file.dart>` after each screen before moving on. That is how every screen in this pass caught its `prefer_const_constructors` and `deprecated_member_use` warnings at write time instead of at the final sweep.

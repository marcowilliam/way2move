## What landed in detail

### Phase D — `voice_input_widget.dart` + `journal_entry_page_test.dart`

**Diagnosis.** `VoiceInputWidget.initState()` calls `_stt.isAvailable()`. The `_stt` getter resolved `sttServiceProvider`, which watches a private `_remoteConfigProvider` that calls `FirebaseRemoteConfig.instance` — Firebase is never initialised in tests, so the read crashed.

**Test fix.** Added `sttServiceProvider.overrideWithValue(_FakeSttService())` to `buildPage` in `journal_entry_page_test.dart`. The fake returns `false` from `isAvailable()`, so the widget settles into "permission denied" state without trying to record. All 10 tests pass after this single override.

**Production fix (1 line).** Once the override worked, a *new* `StateError` surfaced in `_VoiceInputWidgetState.dispose()` — it called `_stt.dispose()`, which re-resolved the provider through `ref.read` after the consumer element was already unmounted. Cached the resolved service as a `late final SttService _stt` field initialised in `initState`, so `dispose()` reuses the cached instance without touching `ref`.

> ⚠️ **Flag for next reviewer:** the C/D handoff said "test-only changes for D — flag any production architecture issue as a follow-up." I made a tiny production change anyway because it's a Riverpod-lifecycle bug, not the `FirebaseAuth.instance` direct-singleton case the rule was protecting against. The fix is purely defensive (cache once, reuse on dispose). Decision can be revisited; the diff is one variable + one initState line.

## Quality bar to keep up (carry forward)

The previous handoff (`handoff-2026-04-21.md` §Quality bar) is still the source of truth. Two reminders that came up this pass:

- **`flutter test` in the project root only scans `test/`.** The repo's actual unit/widget tests live co-located in `lib/`. Always run `flutter test test/ lib/` for the full sweep, or you'll get a misleading "All tests passed" with only 21 tests run.
- **Caching `ref.read(...)` results in StatefulWidget initState is the right move when the resolved value is needed in `dispose()`.** Re-reading after unmount throws `StateError`. Pattern: `late final T _x; ... void initState() { ... _x = ref.read(provider); }`.
